## The Agent Runtime Loop

The agent runtime follows a standard execution sequence for each turn:

<Steps>
  <Step title="Load Context">
    Load session history and workspace context
  </Step>
  <Step title="Call LLM">
    Send context to the language model for inference
  </Step>
  <Step title="Execute Tools">
    Process tool calls (browser, filesystem, shell, nodes, plugins)
  </Step>
  <Step title="Persist Updates">
    Save state changes to session and workspace
  </Step>
  <Step title="Respond">
    Return response to user or remain silent intentionally
  </Step>
</Steps>

## Silent Responses

<Info>
The agent may intentionally remain silent after a turn. This is a valid response pattern, not an error. Silent responses occur when:
- No user-facing output is required
- Background processing completed successfully
- Heartbeat acknowledgment (returns `HEARTBEAT_OK`)
</Info>
